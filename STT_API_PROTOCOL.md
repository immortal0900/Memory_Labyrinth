# STT API 
BASE URL: http://100.100.53.32:8000   

**두개의 API는 서로 같은 기능 입니다.**    
클라이언트 환경에서 사용하기 편한걸로 써주세요.
- 주의사항: 16khz 로 맞춰서 보내주세요. 
- (사용 모델이 내부적으로 16khz로 리샘플링을 합니다. 48khz를 사용하는 경우 불필요한 리샘플링이 발생해서 레이턴시가 느려집니다.)


## 1. wav 전달 버전 
POST http://100.100.53.32:8000/stt    
1. multipart 형식입니다.   
2. file 이라는 이름으로 .wav(16kHz 으로..) 파일을 보내주시면 됩니다. 
3. (언리얼 사용시) 마이크 녹음 -> wav로 변환 -> api 사용 

## 2. 바이너리 스트림 버전
POST http://100.100.53.32:8000/stt_pcm   
1. application/octet-stream 형태입니다.  
2. Body에 byte[] 바이트 배열을 그대로 넣어 보내주시면 됩니다.
3. (언리얼 사용시) 마이크 녹음 → raw 오디오 바이트(int16, 16kHz, mono) 누적 → api 사용


## 응답 형식
둘 다 같은 기능이기 때문에 같은 데이터를 리턴 합니다. 

```
{
    "transferText": " 몬스터들은 뭐야? 좀 도와줘 봐."
```
