# 문서 디렉토리

이 폴더는 프로젝트의 **핵심 기술 문서**를 관리합니다.

## 파일 구조

```
docs/
├── README.md          # 이 파일
└── 핵심기술.md        # 핵심 기술 정리 문서 (Cursor 명령어로 생성/관리)
```

## 핵심기술.md 생성 방법

### 초기 생성

프로젝트에서 처음으로 핵심 기술을 정리할 때:

```
/Main features [키워드]
```

**예시:**
- `/Main features npc` - NPC 관련 핵심 기술 정리
- `/Main features memory` - 메모리 시스템 핵심 기술 정리
- `/Main features all` - 프로젝트 전체 핵심 기술 정리

### 문서 업데이트

기존 문서에 기능을 추가하거나 수정할 때:

```
/Main features-sync [기능명]
```

**예시:**
- `/Main features-sync 하이브리드검색` - 하이브리드 검색 섹션 업데이트
- `/Main features-sync 의도분류` - 의도 분류 섹션 추가 또는 업데이트

## 문서 템플릿 구조

`핵심기술.md`는 다음과 같은 구조로 작성됩니다:

```markdown
---
생성날짜: YYYY-MM-DD HH:mm
마지막수정날짜: YYYY-MM-DD-요일 HH:mm
tags:
- 핵심기술
- 포트폴리오
type:
- 기술문서
---

# [키워드] 핵심 기술 정리

---

## 1. [기술명 1]

### 선택 이유

|-----------------|------------|
| ... | ... |

### 구현 방법
- **구현 파일**: `path/to/file.py`
- **핵심 로직**: ...
- **참고 문서**: [기술명](URL)

- (기능 1)

```

## 작성 원칙

1. **차별성**: 일반적인 구현과 다른 독자적인 기술만 포함
2. **검증 가능성**: 모든 기술은 실제 구현 파일 경로 포함
3. **출처 명시**: 외부 기술/라이브러리는 공식 문서 링크 필수
4. **초보자 관점**: 코드 없이도 핵심 아이디어를 이해할 수 있게 작성
5. **포트폴리오**: 면접관이나 채용 담당자가 읽어도 명확하게 이해 가능
